# Git Guidelines

## Branches
-   `main`: Production-ready code and approved features.

## Commits
-   Format: `feat: ...`, `fix: ...`, `docs: ...`, `chore: ...`
-   Tone: Imperative ("Add feature", not "Added feature").

## PRs
-   Description must link to relevant User Request or Ticket.
-   Must include "Test Plan" section.
